## 3. Текущее состояние сайта (исходные данные)

### 3.1 Структура и разделы

На текущем сайте присутствуют:

- **Главная:** блоки «Миграционная помощь в Москве и МО», услуги для физлиц и организаций, призывы к действию (заявка/консультация), ссылки на госсервисы, блок FAQ.  
- **Услуги:** РВП, ВНЖ, Гражданство, Разрешение на работу/виза в РФ, Отмена депортации.  
- **Информация:** статьи, вопросы и ответы, новости; подразделы по темам (гражданство в общем/упрощённом порядке, по браку, по детям и т.д.).  
- **Контакты:** адреса (Подольск, Новоивановское), телефоны, форма «Бесплатная консультация» / «Заказать звонок».  

### 3.2 Ключевые элементы

- Шапка: логотип, телефон, кнопка «Бесплатная консультация», меню (услуги, информация, контакты).  
- Формы: заявка на консультацию (имя, телефон), согласие на обработку персональных данных.  
- Блок ссылок на госсервисы: проверка готовности РВП/ВНЖ, ФССП, патент, ИНН, запрет въезда, запись в Сахарово и т.д.  
- Футер: адреса, телефоны, повтор меню, соцсети, юридические данные (ООО «Трудовой консультант», ИНН).  

### 3.3 Ограничения и проблемы (текущие)

- *Перечислить известные проблемы: устаревшая вёрстка, тяжёлые скрипты, неудобное меню на мобильных, медленная загрузка и т.д.*

## 4. Требования к новой вёрстке

### 4.1 Адаптивность

- Упор на мобильную версию.
- Таблицы и длинные списки: горизонтальный скролл или упрощённое отображение на малых экранах.  

### 4.3 SEO

- Корректные заголовки H1–H6 по страницам; один H1 на страницу.  
- Мета-теги (title, description) — управляемые при передаче на CMS.  
- Микроразметка при необходимости (Organization, LocalBusiness, FAQPage и т.д.) — *уточнить.*  
- Сохранение или улучшение текущей структуры URL.  

### 4.4 Производительность

- Оптимизация изображений (форматы, размеры, lazy load).  
- Минимизация блокирующего JS/CSS; отложенная загрузка некритичного.  
- Целевые показатели: *указать LCP, FID/INP, CLS по согласованию.*  

### 4.5 Браузеры и устройства

- Поддержка: *например, последние 2 версии Chrome, Firefox, Safari, Edge; iOS Safari, Chrome Android.*

### 6.1 Шаблон страницы блога (чек-лист)

**Эталон:** [migra-pro.com/check-list/programma-repatriacii](https://migra-pro.com/check-list/programma-repatriacii)

Шаблон используется для пошаговых гайдов и чек-листов (например, программа репатриации, сбор документов на РВП/ВНЖ). При переверстке сохранить логику и состав блоков.

| Элемент | Описание |
|--------|----------|
| **Хлебные крошки** | Цепочка: Главная → раздел (напр. «Чек-листы») → название статьи. |
| **Заголовок H1** | Название материала (одна строка). |
| **Подзаголовок / лид** | Краткое описание цели чек-листа (1–2 предложения). |
| **Оглавление (содержание)** | Список этапов с якорными ссылками на секции (на мобильных — по желанию сворачиваемый блок). |
| **Секции этапов** | Каждый этап — отдельная секция с заголовком (H2). Внутри: подзаголовки (H3), нумерованные шаги, списки документов, сроки, пояснения. |
| **Врезки «Важно» / «Обратите внимание»** | Визуально выделенные блоки для критичной информации внутри этапа. |
| **Блоки с цифрами (сроки, суммы)** | Кратко выделенные показатели (например: «60 дней», «5000 ₽») рядом с текстом. |
| **Кнопки/ссылки действий** | «Лично в МВД», «Через Госуслуги» и т.п. — при необходимости отдельное оформление. |
| **Боковая колонка (или блок внизу на мобильных)** | «Полезная информация», «Есть вопрос по этой теме?» — текст + CTA (форма заявки / кнопка «Заказать консультацию»). |
| **Повтор оглавления** | В конце статьи — дублирование навигации по этапам (для длинных страниц). |
| **CTA в конце** | Блок «Не знаете с чего начать?» / «Оставить заявку на бесплатную консультацию». |

Вёрстка должна корректно отображать длинные списки, вложенные списки документов и таблицы (где они есть); на мобильных — читаемые отступы и переносы.
